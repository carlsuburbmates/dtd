# Standards Set

This directory contains the normative product and launch standards set.

These documents are specification-first.
They define what the platform must be, how it must be built, what conditions must be true before launch, and what must be audited after build.

They do not record progress, completion state, or current session status.

## Documents

1. `SSOT.md`
- Canonical blueprint and operating specification.

2. `BUILD_CHECKLIST.md`
- Execution checklist for implementation and assembly.

3. `LAUNCH_GATE.md`
- Required criteria for launch approval.

4. `INTEGRITY_AUDIT.md`
- Post-build verification and audit checklist.

## Rule

If a status-tracking or historical document conflicts with this standards set on intended product behavior, operating model, or launch-quality expectations, this standards set should be treated as the normative target and the status-tracking documents should be updated separately.
